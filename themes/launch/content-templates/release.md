short sections below, then delete these instructions and set
`draft: false`. A changelog is proof of life — short and regular beats
long and rare.*

*First paragraph: what changed, concretely. Name the screen or the API
field. One screenshot here is worth three paragraphs —
`![What changed](/media/release.png)`.*

*Second paragraph: why it matters — the problem a user had that this
removes. If it was a bug, say plainly what was wrong and who was
affected; owning it reads as confidence, because it is.*
