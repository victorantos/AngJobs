I run [Copperline](/projects/), a one-person uptime and status-page service, and I keep this blog as the logbook: what broke, what shipped, what I'd do differently. Some posts are deep technical dives with real query plans in them. Some are essays about running a business with a team size of one. Both are written for the same reader — a developer who might do this someday.

Start with the [blog](/blog/), skim [what I've built](/projects/), or see [what I work with](/uses/). If you only read one thing, read [the Postgres queue post](/blog/a-postgres-queue-two-years-in/) — it's the most useful thing here.

New posts land roughly twice a month. There's an [RSS feed](/blog/rss.xml), because of course there is.
