Mixpeek - Senior Infrastructure Engineer, Applied AI Engineer - NYC or REMOTE (US) - Full-time - <a href="https:&#x2F;&#x2F;mixpeek.com" rel="nofollow">https:&#x2F;&#x2F;mixpeek.com</a>

Mixpeek is multimodal AI infrastructure — we turn unstructured content (video, images, audio, documents) into searchable, programmable assets through a unified API. Think of us as the data infrastructure layer that sits between your raw media and your AI applications. If you&#x27;ve ever tried to build a pipeline that extracts features from video, makes them retrievable, and serves them at scale — you know this is a 12-18 month infrastructure project. We compress that to an afternoon.

We&#x27;re currently building MVS (Mixpeek Vector Store) — a distributed vector database built on Ray + S3, designed to handle 100B+ vectors at a fraction of the cost of existing solutions. Architecture details: shard-level WAL, LIRE-based adaptive search, replica sets, and agent-native query primitives. If you&#x27;ve ever wanted to rethink how vector search works from the storage layer up, this is that project.

Some things we&#x27;re shipping right now:

- IP safety for media &amp; sports — our copyright detection platform (<a href="https:&#x2F;&#x2F;copyright.mixpeek.com" rel="nofollow">https:&#x2F;&#x2F;copyright.mixpeek.com</a>) helps brands and leagues detect unauthorized use of visual IP at scale. We&#x27;re working with partners in the media&#x2F;sports ecosystem including Backblaze for storage-native integration.

- Healthcare pipelines — multimodal extraction for clinical trial recruitment and SNF&#x2F;MDS coding workflows, working with enterprise partners in the space.

- Ad verification — we contribute to the IAB Tech Lab ARTF working group and power contextual intelligence for ad safety.

Our core primitives: feature extractors, retrievers, taxonomies, clusters. Decompose with extractors, recompose with retrievers. Docs: <a href="https:&#x2F;&#x2F;docs.mixpeek.com" rel="nofollow">https:&#x2F;&#x2F;docs.mixpeek.com</a>

Stack: Python, Ray, S3, FastAPI, React&#x2F;TypeScript. We also maintain amux [<a href="https:&#x2F;&#x2F;github.com&#x2F;mixpeek&#x2F;amux" rel="nofollow">https:&#x2F;&#x2F;github.com&#x2F;mixpeek&#x2F;amux</a>], an open-source tmux multiplexer for running parallel Claude Code agent sessions — if you&#x27;re into agentic dev workflows, check it out.

I&#x27;m Ethan (founder&#x2F;CEO, previously led search at MongoDB). Small team, high ownership, real problems. We&#x27;re preparing for NAB Show next week and scaling enterprise pipeline work across healthcare, adtech, and media.
